# creds

Personal credential manager for macOS. One place for every API key, stored securely in the macOS Keychain with a full-screen TUI and a headless CLI for scripting.

```
┌─────────────────────────────┬──────────────────────────────────────────────┐
│  Filter: _                  │  Anthropic Claude  [P]  🔒 masked            │
│                             │                                               │
│  ✓  Anthropic Claude        │  API Key                                      │
│  ✓  Cerebras                │    Status:   set                              │
│  ✓  Mistral                 │    Value:    sk-a••••••••t-1                  │
│  -  OpenAI                  │    Env var:  ANTHROPIC_API_KEY                │
│  -  Google API Key          │    Updated:  2 days ago                       │
│  -  Notion                  │                                               │
│  -  GitHub                  │  e=Edit  r=Rotate  f=Flag  c=Copy  v=Reveal  │
└─────────────────────────────┴──────────────────────────────────────────────┘
```

## Security model

- **Keychain-backed** — every credential is stored in the macOS Keychain under the service name `io.creds.store`, the same AES-256-GCM encrypted store used by Safari, iCloud Keychain, and system apps
- **Locked at rest** — the Keychain is inaccessible when your screen is locked or the machine is asleep
- **Never written to disk** — `creds env` emits `export KEY=value` directly to stdout; values go into shell memory only. Any `.env` file generated by scripts is ephemeral (permissions 600, regenerated each run, gitignored)
- **Shell-safe output** — single quotes in values are properly escaped so `eval $(creds env)` cannot execute arbitrary commands
- **Clipboard auto-clear** — `c` copies a value to the clipboard and clears it after 30 seconds

## Requirements

- macOS 12+
- Python 3.11+

## Installation

```bash
unzip creds.zip && cd creds && bash install.sh
```

The installer wizard will:
1. Check prerequisites
2. Create a virtual environment and install the package
3. Symlink `creds` to `~/.local/bin/creds`
4. Scan your existing Keychain for legacy API keys and offer to migrate them
5. Optionally set up monthly rotation reminders via [ntfy](https://ntfy.sh)
6. Show a full audit of your credential status

## Usage

### TUI (full-screen dashboard)

```bash
creds
```

| Key | Action |
|-----|--------|
| `a` | Add credential |
| `e` | Edit selected |
| `r` | Rotate selected |
| `f` | Flag as suspect |
| `c` | Copy value to clipboard (clears in 30s) |
| `v` | Reveal / hide value |
| `m` | Migrate legacy keys |
| `W` | Filter work credentials |
| `P` | Filter personal credentials |
| `/` | Search |
| `?` | Help |
| `q` | Quit |

### CLI

```bash
# Get a single value (pipe-friendly, no newline)
creds get anthropic

# Load all keys into the current shell
eval $(creds env)

# Load only work credentials
eval $(creds env --work)

# Show status of all credentials
creds audit

# Show only missing credentials
creds audit --missing

# Show only credentials overdue for rotation
creds audit --stale

# Add or update a credential interactively
creds add anthropic

# Set non-interactively (for scripting — prefer creds add for secrets)
creds set anthropic VALUE

# Check if a credential is set (exit 0 = set, exit 1 = missing)
creds check anthropic

# Flag a credential as suspect (e.g. after a 401 error)
creds flag anthropic

# Migrate legacy Keychain entries
creds migrate --dry-run   # preview
creds migrate --yes       # migrate all
creds migrate             # interactive
```

### In shell scripts

Source the `require-cred` helper to auto-prompt for missing credentials:

```bash
source ~/.local/bin/require-cred

require_cred anthropic ANTHROPIC_API_KEY required
require_cred notion    NOTION_API_KEY    optional
```

If a required credential is missing, it launches `creds add <service>` and retries.

## Credential registry

The default registry includes commonly used services. Edit `creds/services.yaml` to add your own:

```yaml
- id: my-service
  label: My Service
  category: API
  context: personal        # personal | work | both
  required: false
  fields:
    - id: api_key
      label: API Key
      env_var: MY_SERVICE_API_KEY
      secret: true
  hint: "Get your key at https://myservice.com/settings"
```

## Rotation reminders

`creds audit --stale` shows credentials approaching or past their rotation window (default: warn at 90 days, overdue at 180 days).

Configure thresholds via the TUI settings or:

```python
from creds.meta import MetaStore
meta = MetaStore()
meta.set_setting("rotation_warn_days", "60")
meta.set_setting("rotation_overdue_days", "120")
```

If you set up an ntfy topic during installation, `creds-rotation-check` runs on the 1st of each month and pushes a notification if any credentials are overdue.
